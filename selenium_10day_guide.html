Introduction to Selenium & Environment Setup

Understand what Selenium is, its architecture, and get your Python + WebDriver environment ready to write your very first automated test.

Beginner Theory + Setup ~2 Hours
What is Selenium?
🌐
Web Automation Tool
Selenium automates browsers just like a real human user — clicking, typing, navigating pages, and verifying results.
🆓
Open Source
Free to use, maintained by the Selenium community. Supports Python, Java, C#, JavaScript, Ruby, and more.
🔁
Cross-Browser
Run the same test on Chrome, Firefox, Edge, Safari without rewriting code. One API, all browsers.
🧪
Testing Powerhouse
Used for functional testing, regression testing, and end-to-end testing of web applications.
Selenium Architecture
🔧 How Selenium Works
Your Python Test Script → Selenium WebDriver API → Browser Driver (ChromeDriver) → Actual Browser (Chrome/Firefox/Edge)
  • Selenium WebDriver — The core component. It directly communicates with the browser via native browser-specific drivers.
  • Browser Drivers — ChromeDriver (Chrome), GeckoDriver (Firefox), EdgeDriver (Edge). Translate Selenium commands to browser actions.
  • WebDriver Protocol — Uses W3C WebDriver standard — a RESTful HTTP API between Selenium and the browser driver.
  • Selenium Grid — Run tests in parallel across multiple machines and browsers simultaneously.
  • Selenium IDE — A browser extension that records and replays test steps (no coding needed).
Installation Steps
  1. Install Python 3.x from python.org — make sure to check "Add to PATH" during installation.
  2. Install Selenium library using pip: pip install selenium
  3. Install WebDriver Manager (auto-manages browser driver versions): pip install webdriver-manager
  4. Install pytest for running tests: pip install pytest
  5. Verify installation by running: python -c "import selenium; print(selenium.__version__)"
💡 Pro Tip
Using webdriver-manager automatically downloads the correct driver for your installed browser version. No manual driver setup needed!

Locator Strategies Overview
🎯 What is a Locator?
A locator tells Selenium which HTML element to interact with. Choosing the right locator makes tests reliable and maintainable.
LocatorBased OnReliabilityUse When
By.IDid attribute⭐⭐⭐⭐⭐Always preferred if ID exists
By.NAMEname attribute⭐⭐⭐⭐Form inputs with name attr
By.CLASS_NAMEclass attribute⭐⭐⭐Single-class elements
By.TAG_NAMEHTML tag⭐⭐Getting all elements of a type
By.LINK_TEXTExact anchor text⭐⭐⭐⭐Exact hyperlink text
By.PARTIAL_LINK_TEXTPartial anchor text⭐⭐⭐Partial hyperlink text
By.CSS_SELECTORCSS rules⭐⭐⭐⭐⭐Complex UI, no ID/Name

XPath — Deep Dive
  • Absolute XPath: Starts from the root /html/body/div/form/input — fragile, breaks if structure changes.
  • Relative XPath: Starts with // — searches entire DOM: //input[@id='email']
  • contains(): Partial match: //button[contains(text(),'Login')]
  • starts-with(): Prefix match: //input[starts-with(@name,'user')]
  • text(): Match by visible text: //h1[text()='Welcome']
  • Axes: Navigate DOM relationships — parent, child, following-sibling, preceding-sibling.
CSS Selector — Deep Dive
PatternExampleDescription
#id#loginBtnSelect by ID
.class.btn-primarySelect by class
taginputSelect by tag
[attr=val]input[type='email']Attribute equals value
[attr*=val]div[class*='nav']Attribute contains value
parent > childform > inputDirect child
parent childform inputAny descendant
:nth-child(n)li:nth-child(2)Nth child element
Best Practices for Locators
  • Priority Order: ID → Name → CSS Selector → XPath. Avoid Tag Name for unique selection.
  • Never use absolute XPath in production tests — one DOM change breaks everything.
  • Use data-testid attributes — Ask developers to add custom attributes like data-testid="login-btn" for test stability.
  • Inspect in Browser: Right-click → Inspect → Use browser DevTools to build and verify locators before adding to code.
  • Keep locators DRY: Store locators in Page Object classes, not hardcoded in every test.

Handling Browser Alerts
  • Simple Alert: Shows a message with OK button only. Use driver.switch_to.alert.accept()
  • Confirmation Alert: Has OK and Cancel. Accept = OK, Dismiss = Cancel.
  • Prompt Alert: Has a text input field. Use alert.send_keys() before accept.
  • WebDriverWait for alerts: Wait until alert is present before switching using EC.alert_is_present()
JavaScript Executor
  • Use driver.execute_script() to run JavaScript directly in the browser page.
  • Useful for: scrolling to elements, clicking hidden elements, changing values not reachable via Selenium.
  • Example: driver.execute_script("arguments[0].click();", element) — JS click bypasses visibility checks.
  • Scroll down: driver.execute_script("window.scrollBy(0, 500);")

Waits in Selenium

Eliminate flaky tests by mastering Implicit Waits, Explicit Waits, and Fluent Waits — the most critical concept for stable automation.

Intermediate Synchronization ~3 Hours
Why Waits are Essential
⚠️ The Flaky Test Problem
Modern web apps use JavaScript to dynamically load content. Without waits, Selenium tries to find elements before they appear — causing NoSuchElementException and intermittent failures called "flaky tests."
⏰
Implicit Wait
Sets a global timeout. Selenium will poll the DOM for up to N seconds for any element before throwing an exception.
🎯
Explicit Wait
Waits for a specific condition on a specific element. Most powerful and recommended approach.
🔄
Fluent Wait
Advanced version of Explicit Wait. Customize polling interval and ignore specific exceptions during polling.
😴
time.sleep() ❌
Hard-coded sleep. Always waits the full duration even if element loads faster. Slows tests and is unreliable.
Expected Conditions (EC)
Expected ConditionDescription
presence_of_element_locatedElement is in DOM (may be hidden)
visibility_of_element_locatedElement is visible on page
element_to_be_clickableElement is visible AND enabled
text_to_be_present_in_elementElement contains specific text
title_containsPage title contains text
title_isPage title equals exact string
url_containsURL contains specific text
alert_is_presentAn alert dialog is open
frame_to_be_available_and_switch_to_itFrame available and switched
staleness_ofElement no longer in DOM
Implicit vs Explicit vs Fluent
FeatureImplicitExplicitFluent
ScopeGlobal (all elements)Specific elementSpecific element
ConditionElement present in DOMAny EC conditionAny EC + custom ignore
Poll IntervalFixed (500ms)Fixed (500ms)Configurable
Ignore ExceptionsNoNoYes (configurable)
RecommendedAvoid mixing with Explicit✅ YesAdvanced scenarios
⚠️ Never Mix Implicit + Explicit Waits
Mixing them causes unpredictable behavior. The waits can compound and create unexpected timeouts. Use one type consistently per test — prefer Explicit Waits.
Implicit Wait
PYTHON
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.chrome.service import Service
from webdriver_manager.chrome import ChromeDriverManager

driver = webdriver.Chrome(service=Service(ChromeDriverManager().install()))

# Selenium will automatically wait up to 10s for any element
element = driver.find_element(By.ID, "dynamic-element")
element.click()

driver.quit()
Explicit Wait with Expected Conditions
PYTHON
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.webdriver.chrome.service import Service
from webdriver_manager.chrome import ChromeDriverManager

driver = webdriver.Chrome(service=Service(ChromeDriverManager().install()))
driver.get("https://the-internet.herokuapp.com/dynamic_loading/1")

# Create WebDriverWait object — 15s max timeout
wait = WebDriverWait(driver, 15)

# Click the Start button
start_btn = wait.until(EC.element_to_be_clickable((By.XPATH, "//button[text()='Start']")))
start_btn.click()

# Wait for the loading to finish (element becomes visible)
hello_msg = wait.until(EC.visibility_of_element_located((By.ID, "finish")))
print("Loaded:", hello_msg.text)

# Wait for URL to change
wait.until(EC.url_contains("success"))

# Wait for page title
wait.until(EC.title_contains("Dynamic"))

# Wait for text in element
wait.until(EC.text_to_be_present_in_element((By.ID, "status"), "Complete"))

driver.quit()
Fluent Wait
PYTHON
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.common.exceptions import NoSuchElementException, StaleElementReferenceException
from datetime import timedelta

# Fluent Wait: max 30 seconds, poll every 2 seconds,
# ignore NoSuchElementException and StaleElementReferenceException
fluent_wait = WebDriverWait(
    driver,
    timeout=30,
    poll_frequency=2,   # check every 2 seconds
    ignored_exceptions=[NoSuchElementException, StaleElementReferenceException]
)

# Use it just like explicit wait
element = fluent_wait.until(
    EC.visibility_of_element_located((By.ID, "ajax-result"))
)
print(element.text)

Select Class for Dropdowns
📋 What is the Select Class?
The Select class in Selenium provides dedicated methods for interacting with HTML <select> elements. Import it from selenium.webdriver.support.ui.
MethodDescription

select_by_value(value)Select by HTML value attribute

deselect_by_visible_text(text)Deselect in multi-select

Checkboxes & Radio Buttons
  • Click to toggle — Use .click() to check/uncheck checkboxes.
  • Verify state — Use .is_selected() to check if a checkbox or radio is currently selected.
  • Conditional click — Check only if not already selected: if not checkbox.is_selected(): checkbox.click()
  • Radio buttons — Only one can be selected at a time. Just .click() the desired radio button.
  • Grouped checkboxes — Use find_elements() to get all checkboxes in a group, then iterate.
Custom Dropdowns (Non-Select)
⚠️ Not All Dropdowns are <select>
Modern frameworks (React, Angular, Vue) often use custom divs/uls for dropdowns. These do NOT use the Select class. Instead: click the dropdown trigger → wait for options to appear → find and click the option.
  • Click the dropdown trigger element (the visible button/div).
  • Wait for the dropdown menu to appear using explicit wait.
  • Find the desired option using XPath with text() or contains().
  • Click the option.
Select Dropdown — All Methods
PYTHON
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import Select
from selenium.webdriver.chrome.service import Service
from webdriver_manager.chrome import ChromeDriverManager

driver = webdriver.Chrome(service=Service(ChromeDriverManager().install()))
driver.get("https://the-internet.herokuapp.com/dropdown")

# Locate the dropdown element and wrap with Select class
dropdown_elem = driver.find_element(By.ID, "dropdown")
dropdown = Select(dropdown_elem)

# Method 1: Select by visible text
dropdown.select_by_visible_text("Option 1")

# Method 2: Select by value attribute
dropdown.select_by_value("2")

# Method 3: Select by index (0-based)
dropdown.select_by_index(1)

# Get all options
all_options = dropdown.options
print(f"Total options: {len(all_options)}")
for option in all_options:
    print(f"  - {option.text} [value={option.get_attribute('value')}]")

# Get currently selected option
selected = dropdown.first_selected_option
print("Selected:", selected.text)

# Multi-select dropdown
multi_dropdown = Select(driver.find_element(By.ID, "multi"))
multi_dropdown.select_by_visible_text("Python")
multi_dropdown.select_by_visible_text("Java")
all_selected = multi_dropdown.all_selected_options
print("Selected:", [opt.text for opt in all_selected])
multi_dropdown.deselect_all()

driver.quit()

Frames, Windows & Tabs

Switch between iFrames, multiple browser windows, and tabs — essential for testing portals, payment gateways, and OAuth flows.

Intermediate Window Management ~2.5 Hours
Understanding iFrames
🖼️ What is an iFrame?
An iFrame (inline frame) is an HTML element that embeds another document within the current page. Examples: embedded videos, payment forms, Google Maps, reCAPTCHA, social media widgets.
  • Selenium operates within a context. By default, it's the main document context.
  • To interact with elements inside an iFrame, you must switch to that frame first.
  • After working inside a frame, switch back to the main document with driver.switch_to.default_content()
  • Nested frames require switching step by step — frame within a frame.
Window / Tab Management
  • window_handles — A list of all open window/tab handle IDs. The first handle is the original window.
  • current_window_handle — Returns the handle of the currently focused window.
  • switch_to.window(handle) — Switch focus to a specific window by handle.
  • New tab via JS — Use driver.execute_script("window.open()") to open a new blank tab.
  • Always store the parent handle — Before clicking a link that opens a new window, save the parent handle to return later.

ActionChains Class
⛓️ What are ActionChains?
ActionChains is a Selenium class that allows chaining of multiple user actions. All actions are queued and executed together when .perform() is called.
MethodDescription
move_to_element(elem)Hover over element (mouse over)
click(elem)Single left click
double_click(elem)Double-click element
context_click(elem)Right-click (context menu)
click_and_hold(elem)Press and hold left mouse button
release()Release mouse button
drag_and_drop(src, target)Drag element from source to target
drag_and_drop_by_offset(src, x, y)Drag by pixel offset
send_keys(keys)Type keys at current focus
key_down(key)Hold down a key
key_up(key)Release a held key
scroll_to_element(elem)Scroll until element is in view
perform()Execute all queued actions
reset_actions()Clear queued actions
When to Use ActionChains
  • Hover menus: Navigation menus that only appear on mouse hover cannot be clicked directly without hovering first.
  • Drag and Drop: Sortable lists, kanban boards, slider controls.
  • Tooltips: Verifying tooltips that appear on hover.
  • Right-click menus: Testing custom context menus in web apps.
  • Keyboard shortcuts: CTRL+C, CTRL+V, CTRL+Z — multi-key combinations.

Page Object Model (POM)

Master the industry-standard POM design pattern — write maintainable, scalable, and reusable Selenium test architectures.

Advanced Design Patterns ~4 Hours
What is Page Object Model?
🏗️ POM Definition
POM is a design pattern that creates a separate class for each web page. Each class contains all locators and methods for that page. Tests use these classes instead of directly interacting with browser elements. This separates test logic from UI interaction logic.
🔧
Maintainability
If a locator changes, update only one Page class — not every test that uses it.
♻️
Reusability
Page classes are reused across multiple test files. Write once, use everywhere.
📖
Readability
Tests read like plain English: login_page.enter_username("tom").click_login(). Clear and understandable.
🧩
Separation of Concerns
Test logic (what to test) is separated from page logic (how to interact). Cleaner codebase.

POM Rules
  • One class per page — Each web page has its own Python class.
  • Locators as class attributes — Define locators at the top of the class.
  • No assertions in page classes — Assertions belong in test files only.
  • Methods return page objects — Actions that navigate to new pages should return the new page's object (method chaining).
  • Base Page class — Common methods (wait, click, type) go in a parent BasePage class.

# pages/base_page.py
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.common.exceptions import TimeoutException

class BasePage:
    def __init__(self, driver, timeout=10):
        self.driver = driver
        self.timeout = timeout
        self.wait = WebDriverWait(driver, timeout)

    def open(self, url):
        self.driver.get(url)
        return self

    def find(self, locator):
        return self.wait.until(EC.presence_of_element_located(locator))

    def click(self, locator):
        self.wait.until(EC.element_to_be_clickable(locator)).click()

    def type_text(self, locator, text):
        element = self.wait.until(EC.visibility_of_element_located(locator))
        element.clear()
        element.send_keys(text)

    def get_text(self, locator):
        return self.wait.until(EC.visibility_of_element_located(locator)).text

    def is_visible(self, locator):
        try:
            self.wait.until(EC.visibility_of_element_located(locator))
            return True
        except TimeoutException:
            return False

    def take_screenshot(self, name):
        self.driver.save_screenshot(f"screenshots/{name}.png")

# pages/login_page.py
from selenium.webdriver.common.by import By
from pages.base_page import BasePage

class LoginPage(BasePage):
    # ─── Locators ──────────────────────────────────
    URL            = "https://the-internet.herokuapp.com/login"
    USERNAME_INPUT = (By.ID, "username")
    PASSWORD_INPUT = (By.ID, "password")
    LOGIN_BTN      = (By.CSS_SELECTOR, "button[type='submit']")
    FLASH_MSG      = (By.CSS_SELECTOR, ".flash")
    SUCCESS_FLASH  = (By.CSS_SELECTOR, ".flash.success")
    ERROR_FLASH    = (By.CSS_SELECTOR, ".flash.error")

    # ─── Page Actions ──────────────────────────────
    def open_login_page(self):
        return self.open(self.URL)

    def enter_username(self, username):
        self.type_text(self.USERNAME_INPUT, username)
        return self   # Return self for method chaining

    def enter_password(self, password):
        self.type_text(self.PASSWORD_INPUT, password)
        return self

    def click_login(self):
        self.click(self.LOGIN_BTN)
        return self

    def login(self, username, password):
        """Complete login action"""
        return self.enter_username(username)\
                   .enter_password(password)\
                   .click_login()

    # ─── Verification Methods ──────────────────────
    def is_login_successful(self):
        return self.is_visible(self.SUCCESS_FLASH)

    def get_flash_message(self):
        return self.get_text(self.FLASH_MSG)
